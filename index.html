modules

Three sketches. One theme: listen, don't shout.

Each one flashes independently — grab the file for the board you've got and go. Full source and wiring notes are in the repo.

$flash wifi-scanner

wifi-scanner

Turns the board into a standalone site-survey tool. Rescans every 10s and prints a sorted table over Serial.

  • RSSI, channel & encryption per network
  • Find the least congested channel
  • Spot dead zones before placing a repeater
$flash rogue-ap-detector

rogue-ap-detector

A tripwire for evil-twin attacks. Alerts when your trusted SSID shows up from an unfamiliar router or with weakened encryption.

  • BSSID-mismatch detection
  • Flags a suspicious drop to open auth
  • Optional buzzer / LED alarm output
$flash captive-portal

captive-portal-manager

Headless WiFi onboarding. Device opens its own hotspot, you pick a network from your phone, credentials save to flash.

  • No hardcoded SSID/password in the sketch
  • Auto-popup setup page on connect
  • Reconnects on its own after that
a note on scope

Everything here only listens.

Every module works off the same standard scan and access-point APIs your phone uses to show you a list of networks. None of them send a deauthentication frame, inject packets, or sniff traffic that isn't addressed to them.

That's a real design constraint, not a disclaimer. It's also why a $3 board and a USB cable are enough to build something genuinely useful: a channel planner, a break-in alarm, a setup wizard — the boring, durable kind of project that still works in a year.

get started

From board to running firmware

01

Install the core

Arduino IDE → Boards Manager → add the ESP8266 or ESP32 package URL.

02

Pick a module

Open the .ino for the project you want from the repo — each folder is self-contained.

03

Flash it

Select your board and port, hit upload, open Serial Monitor at 115200 baud.

04

Read the output

Every module talks back over Serial — that's your interface, no app required.
